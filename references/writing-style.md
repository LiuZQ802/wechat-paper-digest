# 写作要求：具体，不让读者猜

核心标准只有一条：**读者扫一眼一句话，要能立刻知道这句话在说什么具体的事**，不需要脑补、不需要猜你到底想说啥。模糊的概括句是默认会被拒收的，除非紧跟着具体内容把它落实了。

## 检查方法

写完 `article.md` 后，逐句过一遍，对每一句问：**如果删掉这句话，读者会损失什么具体信息？** 如果答案是"没有损失，这句话只是营造了一种'厉害/重要/有效'的语气"，这句话就要么删掉，要么补上让它具体的内容。

## 典型的"模糊空话"——尽量别这么写

这些词本身不是禁用词，但**单独出现、后面不跟具体内容**时几乎总是在说"我也不知道具体是什么，但听起来很厉害"：

- "显著提升/显著改善" → 提升到多少？从多少到多少？
- "有效解决/有效降低" → 降低了多少？怎么做到的？
- "在一定程度上/某种程度上" → 到底是什么程度，能不能换成数字或者范围？
- "表现良好/效果不错" → 用什么指标衡量的，数值是多少？
- "具有重要意义/具有重要价值" → 意义具体是什么？说出那个意义本身，而不是说"有意义"这件事。
- "新颖的方法/创新性的设计" → 新在哪？跟旧方法具体差在哪一步？
- "多个场景/多种情况下" → 哪几个场景？能不能直接点名？
- "很朴素/很实在/很扎实"这类对实验设计本身的主观评价词 → 直接描述这个实验设计是什么（比如对比了哪两种方式、跑了几组、每组多少样本），不需要先评价一句"朴素""实在"

## 对照例子

| 模糊（不要这样） | 具体（这样写） |
|---|---|
| 该方法效率更高 | 推理只需29.5分钟、花费0.154美元，是GPT-4o（9.12美元）的约1/60 |
| 模型表现有显著提升 | F1从0.3483提升到0.4487，提升了0.1004 |
| 在多个语料上都表现良好 | 在Weibo、CLUENER、人民日报三个语料上都优于随机采样基线 |
| 作者提出了一种新颖的采样策略 | 过去的采样方法只看模型当前这一轮的预测，这篇论文新增了"看历史多轮预测有多不稳定"这一维度 |
| 这一发现具有重要意义 | （直接说发现本身是什么，不需要再加一句"这很重要"） |
| 该框架综合考虑了多种因素 | 框架同时用了三个指标：温度A、压力B、湿度C（具体点名是哪几个） |

## 适用范围

- **内容导读**、各小节正文、图说，全部按这个标准检查。
- 数字、专有名词、论文里的具体例子/案例，能用就用，不要为了"读起来顺"而磨掉细节。
- 这不等于句子要写得啰嗦——具体和简洁不矛盾，"F1从0.35到0.45"比"模型效果有所改善但仍有提升空间"短得多，但信息量大得多。
- 比喻、生动的小标题（参考 format-rules.md 第6条的鼓励）不在禁止之列，但比喻之后要落回具体内容，不能让比喻代替信息。

## 用词：可以生动，不能网络化/口水化

鼓励小标题有趣、比喻贴切，但要分清"生动"和"网络流行语/大白话"的区别：

- **避免**网络梗/流行语式表达：如"反复横跳"、"真香"、"绝绝子"、"拿捏"——这类词读起来轻佻，跟科学传播的语境不搭。
- **避免**过度口语化的吐槽式形容：如"苦活"、"难啃"、"卷"、"硬核"单独使用——能换成准确描述就换成准确描述（"苦活" → 直接说"标注工作耗时耗力，需要逐字处理"）。
- **可以用**成语、书面语里的生动表达（"对症下药"、"釜底抽薪"、"水落石出"之类），这些是规范书面语，不是网络流行语，达意且不轻佻。
- 判断标准：把这个词单独拎出来，如果是从短视频/弹幕/社交媒体梗图里学来的，不用；如果是词典里就有、写进正式文章不会显得格格不入的，可以用。

## 连贯性：章节之间要有桥梁句，不能是孤立的知识点堆砌

每一节衔接处，读者应该能看出"为什么读完上一节、紧接着要读这一节"，而不是感觉在读几个互不相关的知识点列表。

- 每一节的**开头或上一节的结尾**，写一句承上启下的话，把上一节留下的问题/结论和这一节要讲的内容连起来（例如："要解决这个问题，论文提出了……"、"光挑对样本还不够，标注本身也要……"）。
- 自查方法：把所有小标题单独抽出来看一遍，应该能读出一条完整的逻辑线（提出问题 → 怎么解决 → 怎么验证 → 验证结果说明什么），而不是一堆并列的、可以任意打乱顺序的标题。
- 内容导读里提前点出文章的逻辑主线（要解决几个问题、对应几个方案），后面每节展开时呼应这条主线，读者读到一半时始终知道"现在讲的是主线里的哪一步"。

## 内容要素：核心科学问题、方法、实验设计、结果、意义都要讲清楚

不要求每一节都展开讲全部要素，但整篇文章读完，这五件事读者都应该能说清楚：

1. **核心科学问题**：这篇论文到底想解决什么问题？现有方法卡在哪？（不是"提出了一个新方法"这种空泛说法，要说清楚旧方法具体哪里不行）
2. **方法**：论文用了什么具体手段解决这个问题？关键机制是什么？
3. **实验设计**：在什么数据/场景上验证的？跟谁比、比了什么指标？（哪怕只用一两句话带过，也不能完全不提——"在三个语料上做了对比实验"比完全不交代实验设计要好）
4. **结果**：验证出来的具体数字/结论是什么？
5. **意义**：这个结果对这个领域/这类问题意味着什么？（不是泛泛的"意义重大"，是说清楚意义具体是什么，参照前面"模糊空话"那条）

写完之后自查：这五件事里，有没有哪一件完全没提到？如果有，是不是该补一句。
